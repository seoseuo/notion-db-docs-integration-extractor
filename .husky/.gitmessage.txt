# 기본 형식
# <type>[optional scope]: <subject>
# └─ type(필수): feat, fix, docs, style, refactor, perf, test, chore, ci, build, revert 중 하나
# └─ scope(선택): 변경된 범위(예: auth, user, db 등)
# └─ subject(필수): 최대 50자, 현재형 명령문, 마침표 금지
#
# <body(선택)>: 상세 설명 (72자마다 줄바꿈 권장)
# <footer(선택)>: Breaking Changes 및 이슈 참조 (Closes, Fixes, Resolves)

####################
# type 종류
# feat     : 새로운 기능 추가
# fix      : 버그 수정
# docs     : 문서 변경
# style    : 코드 포맷팅, 세미콜론 누락 등 (로직 영향 없음)
# refactor : 코드 리팩토링 (동작 변경 없음)
# perf     : 성능 개선
# test     : 테스트 코드 추가/수정
# chore    : 설정, 빌드, 패키지 관리 등 소스코드 외 변경
# ci       : CI/CD 관련 변경
# build    : 빌드 시스템 변경
# revert   : 이전 커밋 되돌리기  
####################

# EXAMPLE
# feat(user): 사용자 관리 기능 추가
#
# 회원 가입, 로그인, 로그아웃 기능 구현
# 사용자 정보 수정, 비밀번호 변경 기능 추가
# 이메일 인증 및 인증 코드 전송 기능 구현
#
# Closes #123
